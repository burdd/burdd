# User Stories

## User Roles

- Customer: external user filing tickets
- Developer: project member working on issues
- Admin: project admin managing developers and sprints

## User Stories

### Customer
1. As a Customer, I want to submit a ticket without creating an account so that I can report issues quickly.
2. As a Customer, I want a tokenized status link so that I can check progress on my ticket later.
3. As a Customer, I want to add a screenshot to my ticket so that developers can reproduce the issue.
4. As a Customer, I want to see the latest public status and last updated time so that I know someone is looking at it.

### Developer
5. As a Developer, I want to view a triage queue of new tickets so that I can decide whether to convert or link them.
6. As a Developer, I want to convert a ticket into an issue so that I can start work quickly.
7. As a Developer, I want to link a ticket to an existing issue so that duplicates are tracked together.
8. As a Developer, I want to update issue statuses (In Queue → In Progress → Code Review → Done).
9) As a Developer, I want to filter issues by sprint/status/assignee so that I can focus on my work.

### Admin
10. As an Admin, I want to create and activate sprints with date validation so that planning windows are clear.
11. As an Admin, I want to add/remove project members and set roles so that access is controlled.
